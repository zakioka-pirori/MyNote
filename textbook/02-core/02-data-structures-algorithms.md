# 第 7 章 データ構造とアルゴリズム

## 学習目標

- 主要なデータ構造（配列、連結リスト、ハッシュ、木、ヒープ、グラフ）の動作と計算量を理解する。
- アルゴリズム設計の 4 大技法（分割統治・動的計画法・貪欲法・バックトラック）を使い分けられる。
- 計算量を漸近記法で正しく解析でき、性能のボトルネックを特定できる。
- グラフアルゴリズム（BFS, DFS, Dijkstra, MST, 強連結成分）を実装できる。

## 7.1 なぜアルゴリズムを学ぶか

「動くコード」と「速く動くコード」「正しく動くコード」「保守できるコード」は別物。アルゴリズムは **同じ問題を桁違いに高速に解く方法** を提供する。データ量が大きくなればなるほど、$O(n^2)$ と $O(n \log n)$ の差は致命的になる。

加えて、アルゴリズムを学ぶと **問題を分解する型** が手に入る。新しい問題に出会ったとき、「これはグラフ問題」「これは DP に帰着」と認識できるようになる。

## 7.2 計算量と漸近記法

入力サイズ $n$ に対する操作数を関数で表す。定数倍と低次項を無視するのが **漸近記法**。

- $O(f(n))$: 上界。$\exists c, n_0$ で $\forall n \geq n_0, T(n) \leq c f(n)$。
- $\Omega(f(n))$: 下界。
- $\Theta(f(n))$: 上下界が一致。

主な複雑度クラス（小さい順）: $O(1) < O(\log n) < O(\sqrt n) < O(n) < O(n \log n) < O(n^2) < O(n^3) < O(2^n) < O(n!)$。

**マスター定理**: 再帰式 $T(n) = aT(n/b) + f(n)$ の閉じた形を与える。たとえばマージソート $T(n) = 2T(n/2) + O(n) = O(n \log n)$。

**償却計算量**: 単一操作が稀に重くても全体平均は軽い場合の解析（動的配列の追加が $O(1)$ 償却）。

## 7.3 線形構造

### 配列 (Array)

連続したメモリ。インデックスアクセス $O(1)$、挿入・削除 $O(n)$。キャッシュ局所性が良い。

### 動的配列 (Dynamic Array)

容量不足時に倍々に拡張。$\text{push}$ は償却 $O(1)$。Python の `list`、Java の `ArrayList`、C++ の `vector`。

### 連結リスト (Linked List)

各ノードがポインタで次を指す。挿入・削除 $O(1)$（位置がわかれば）、ランダムアクセス $O(n)$。実務では配列のほうが速いことが多い（キャッシュ）。

### スタック・キュー・デック

- スタック (LIFO): 関数呼び出し、式評価、DFS。
- キュー (FIFO): タスクキュー、BFS。
- デック (両端): スライディングウィンドウ最大値。

### 連想配列・集合

- ハッシュテーブル: 平均 $O(1)$、最悪 $O(n)$（衝突）。
  - 衝突解決: チェイン法、オープンアドレス法（線形・二次・ダブルハッシュ）。
  - 重要パラメータ: ロードファクタ、リハッシュ。
- 平衡二分探索木 (Red-Black, AVL): すべての操作 $O(\log n)$、順序付き走査が可能。

## 7.4 木構造

### 用語

- 根、葉、深さ、高さ、子、親、兄弟。
- 二分木: 各ノードの子が高々 2。
- 完全二分木 / 完全充填木 / バランス木。

### 二分探索木 (BST)

左部分木 < ノード < 右部分木。検索・挿入・削除 $O(h)$（$h$ は高さ）。最悪偏ると $O(n)$。

### 平衡木

- **AVL 木**: 各ノードで左右の高さ差が高々 1。挿入時に回転で復元。
- **赤黒木**: 5 つの規則で近似的にバランス。Linux カーネルや Java TreeMap で使用。
- **B 木 / B+ 木**: 子の数が多い木。ディスク I/O の少なさが利点で、データベースインデックスやファイルシステムで定番。

### ヒープ

完全二分木 + ヒープ性質（親 ≤ 子 が最小ヒープ）。配列で実装、$\text{push}/\text{pop}$ が $O(\log n)$。優先度付きキュー、Dijkstra、トップ K に使う。

### Trie (基数木)

文字列をエッジに持つ木。プレフィックス検索 $O(L)$（$L$ は文字列長）。オートコンプリート、IP ルーティング、辞書。

### Union-Find (素集合森)

要素のグループ化。`find` と `union` を経路圧縮 + ランクで行うと、ほぼ $O(\alpha(n))$（実用上は $O(1)$）。MST、グラフの連結性、Kruskal 法。

## 7.5 ハッシュ深掘り

### ハッシュ関数の性質

- 一様分布
- 衝突困難（暗号用途では強衝突困難）
- 計算が速い

非暗号: MurmurHash, xxHash, FNV。
暗号用途: SHA-256, BLAKE3。

### ブルームフィルタ

「集合に属するかも」を高速・省メモリで判定する確率的データ構造。偽陽性あり、偽陰性なし。CDN, データベース、ネットワークで使われる。

### Count-Min Sketch / HyperLogLog

ストリームデータの頻度推定とユニーク数推定。大規模データ解析の基本ツール。

## 7.6 ソート

### 比較ソート

| 名前 | 平均 | 最悪 | 安定 | 備考 |
|---|---|---|---|---|
| 挿入ソート | $O(n^2)$ | $O(n^2)$ | 安定 | ほぼ整列済みで速い |
| マージソート | $O(n \log n)$ | $O(n \log n)$ | 安定 | 外部ソート向き |
| クイックソート | $O(n \log n)$ | $O(n^2)$ | 不安定 | 実用最速、ピボット選択が肝 |
| ヒープソート | $O(n \log n)$ | $O(n \log n)$ | 不安定 | in-place |

比較ソートの下界は $\Omega(n \log n)$（決定木の深さから）。

### 線形時間ソート

- 計数ソート: 値の範囲が小さいとき $O(n + k)$。
- 基数ソート: 桁ごとに計数。$O(d(n + b))$。
- バケットソート: 一様分布なら $O(n)$。

### Timsort

挿入 + マージのハイブリッド。Python の `sorted`、Java の `Arrays.sort` などで採用。

## 7.7 探索

- 線形探索 $O(n)$。
- 二分探索 $O(\log n)$（整列済み配列）。境界条件（左閉右開）に注意。
- 三分探索: 単峰関数の最大値探索。

## 7.8 グラフアルゴリズム

### 表現

- 隣接行列 $O(V^2)$ 空間、辺の存在確認 $O(1)$。
- 隣接リスト $O(V + E)$ 空間、列挙が速い。

### 探索

- **BFS** (Breadth-First Search): キューで層ごと探索。重みなしの最短路、レベル分け。
- **DFS** (Depth-First Search): スタックまたは再帰。連結成分、トポロジカルソート、橋・関節点。

### 最短路

- **Dijkstra**: 非負重み、$O((V + E) \log V)$ with ヒープ。
- **Bellman-Ford**: 負重み可、$O(VE)$、負閉路検出。
- **Floyd-Warshall**: 全点対、$O(V^3)$、DP。
- **A\***: ヒューリスティック付き Dijkstra。経路探索 (ゲーム, 地図)。

### 最小全域木 (MST)

連結重み付きグラフの全頂点を結ぶ最小重み木。

- **Kruskal**: 辺を重み順、Union-Find。$O(E \log E)$。
- **Prim**: 頂点を貪欲に追加、ヒープ。$O((V+E) \log V)$。

### 強連結成分 (SCC)

有向グラフで「互いに到達可能」な極大集合。Kosaraju 法（DFS 2 回）または Tarjan 法。コンパイラ最適化、Web リンク解析。

### ネットワークフロー

- 最大流問題: Ford-Fulkerson, Edmonds-Karp, Dinic。
- 最大流 = 最小カット定理。
- 二部マッチング、課題割当、画像セグメンテーションへ応用。

## 7.9 アルゴリズム設計技法

### 分割統治 (Divide and Conquer)

問題を小さく分けて解き、結合する。マージソート、クイックソート、FFT、最近点対、Strassen 行列乗算。

### 動的計画法 (DP)

部分問題の解を保存して再利用。最適部分構造 + 部分問題の重複が条件。

例題:
- ナップサック問題（0/1, 完全）
- 最長共通部分列 (LCS)
- 編集距離（Levenshtein）
- 最長増加部分列 (LIS)
- 区間 DP（行列連鎖積）
- ビット DP（巡回セールスマン）

設計手順:
1. 状態を定義（$dp[i][j]$ が何か）
2. 遷移を立式
3. 初期値・順序を決める
4. 答えを取り出す

メモ化（top-down）と表埋め（bottom-up）は同等。

### 貪欲法 (Greedy)

各ステップで局所最適を選ぶ。最適性の証明（交換論法・Matroid）が肝。例: Huffman 符号、最小全域木、活動選択、コインの両替（特定の体系）。

### バックトラック

候補を試して失敗したら戻る。N クイーン、数独、ハミルトン路。枝刈りで高速化。

### 分枝限定 (Branch and Bound)

バックトラック + 上下界での枝刈り。整数計画、TSP。

## 7.10 文字列アルゴリズム

- KMP, Boyer-Moore: パターンマッチ。
- Z アルゴリズム、サフィックスオートマトン、サフィックス配列。
- ローリングハッシュ、Rabin-Karp。
- 編集距離、最長共通部分列。
- Aho-Corasick: 複数パターン。
- 正規表現の NFA / DFA 構築（第 13 章へ）。

## 7.11 数値・幾何アルゴリズム

- 高速冪、ユークリッドの互除法、拡張ユークリッド、modular inverse。
- 篩 (Eratosthenes, 線形)。
- 行列累乗で漸化式 $O(\log n)$。
- 凸包（Graham scan, Andrew）、線分交差、最近点対。
- セグメント木、フェニック木 (BIT)、永続データ構造。

## 7.12 並列・近似・確率的アルゴリズム

- 並列ソート、並列スキャン、MapReduce。
- 近似アルゴリズム (NP 困難に対する 2-近似など)。
- ランダム化アルゴリズム: クイックソートのピボット、ミラーラビン素数判定、スキップリスト。

## 7.13 計算量の限界

- P / NP / NP 完全（第 8 章で詳細）。
- 「速いアルゴリズムが存在するか」が決着していない問題が多い。
- 既知の難問は近似・ヒューリスティック・SAT/SMT・ILP に委ねる。

## 7.14 演習

1. 動的配列の `push` が償却 $O(1)$ である理由を、$2^k$ 倍拡張で説明せよ。
2. 赤黒木の挿入で行う 5 つのケースを列挙せよ。
3. Dijkstra が負の重みで誤動作する例を 1 つ挙げよ。
4. ナップサック問題の DP 解の状態と遷移を書き、$O(nW)$ で解けることを確認せよ。
5. 文字列 `ABCBDAB` と `BDCAB` の LCS を求めよ。
6. Union-Find の経路圧縮 + ランクの実装を書け。
7. 二分探索で「`a[i] >= x` を満たす最小の $i$」を求める関数の境界条件を厳密に書け。
8. 隣接リスト表現でグラフを定義し、SCC を Kosaraju 法で求める疑似コードを書け。

## 7.15 まとめ

データ構造は「データの並べ方」、アルゴリズムは「データの処理手順」。両者は不可分で、適切な構造を選べばアルゴリズムが自然に決まる。本章で扱った構造と手法は、競技プログラミングだけでなく、データベースのインデックス、コンパイラの最適化、ネットワークのルーティング、機械学習の前処理、すべての CS 分野で再登場する。**「これは BFS だ」「これは DP だ」と即座に分類できる嗅覚** を、繰り返し問題を解いて身につけよ。
