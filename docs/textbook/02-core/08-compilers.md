# 第 13 章 コンパイラと言語処理系

## 学習目標

- ソースコードがどう機械語に変換されるかをパイプラインで説明できる。
- 字句解析・構文解析・意味解析・最適化・コード生成・ランタイムを一通り実装できる程度に理解する。
- 型システム、ガベージコレクション、JIT などの現代的トピックを把握する。

## 13.1 なぜコンパイラを学ぶか

コンパイラは「言語を別の言語に変換する」万能技術。SQL の最適化、正規表現の DFA 構築、シェーダのコンパイル、機械学習の計算グラフ最適化、ビルドツール（webpack, Babel）はすべてミニコンパイラ。**1 つコンパイラを書くと、CS の他分野（オートマトン・データ構造・グラフ・最適化）が一気に繋がる**。

## 13.2 コンパイラの全体像

```
Source → Lexer → Parser → Semantic Analysis → IR → Optimization → Codegen → Target
```

- **フロントエンド**: 字句・構文・意味解析。
- **ミドルエンド**: 中間表現と最適化。
- **バックエンド**: コード生成、レジスタ割付。

LLVM はフロント/バックを分離した設計の代表で、Clang, Rust, Swift など多くの言語で使われている。

## 13.3 字句解析 (Lexical Analysis)

ソースを **トークン列** に変換する。

```
int x = 42;
→ [INT] [IDENT(x)] [ASSIGN] [NUMBER(42)] [SEMI]
```

### 実装

- 正規表現 → NFA（Thompson 構成法）→ DFA（部分集合構成）→ 最小化。
- ツール: `lex`, `flex`, ANTLR の lexer。
- 手書きでも実装は単純（バッファ + 状態機械）。

### 注意点

- 最長一致 (`==` を `=`+`=` にしない)。
- キーワード vs 識別子の区別はテーブル参照。
- 数値リテラル、文字列リテラル、コメントの扱い。

## 13.4 構文解析 (Parsing)

トークン列から **抽象構文木 (AST)** を作る。

### 文脈自由文法

第 8 章で扱った CFG が基礎。曖昧文法を避けるか、優先度・結合性で消す（演算子優先順位）。

### 解析法

#### LL(k) — 下向き

左から右、左端導出、先読み k トークン。

- **再帰下降 (Recursive Descent)**: 各非終端に関数を 1 つ。手書きしやすい。
- **LL(1) パーサジェネレータ**: 予測表を構築。

```c
expr  := term { ('+' | '-') term }
term  := factor { ('*' | '/') factor }
factor := NUMBER | '(' expr ')'
```

左再帰は除去する必要がある（`A → A α | β` → `A → β A'; A' → α A' | ε`）。

#### LR(k) — 上向き

シフト・還元。手書き不向きだが、より広いクラスを扱える。
- LR(0), SLR, LALR(1), LR(1)。
- `yacc`/`bison` が LALR(1)。

#### PEG (Parsing Expression Grammar)

順序付きの選択。`packrat` パーサで線形時間。曖昧性が起きにくいが、左再帰は工夫が必要。

#### GLR / Earley

任意の CFG を扱える、最悪 $O(n^3)$。自然言語処理で利用。

### エラー回復

パニックモード（同期トークンまで読み飛ばす）、フレーズレベルなど。良いエラーメッセージは生産性に直結。

## 13.5 意味解析

AST に対する追加検査:
- 名前解決（変数・型・関数のスコープを解決）。
- 型検査（次節）。
- 制御フロー解析（到達不能コード、未初期化変数）。
- シンボルテーブルの構築。

スコープは入れ子で、シンボルテーブルもスタック的に管理する。

## 13.6 型システム

### 単純型

- プリミティブ: int, float, bool, char。
- 合成: 関数型 `A -> B`、タプル、レコード、配列。

### 型推論

明示しなくても型が決まる仕組み。
- **Hindley-Milner**: ML, OCaml, Haskell。多相を扱える、効率的アルゴリズム W。
- 局所型推論: Java 10 の `var`、C# の `var`、TypeScript。

### 多相

- パラメトリック多相（ジェネリクス）: `List<T>`。
- サブタイプ多相: `Animal` のメソッドを `Dog` で。
- アドホック多相（オーバーロード、型クラス）。

### 高度な型

- 依存型 (dependent type): 値が型に現れる (Coq, Idris, Lean)。
- 線形型 (linear type): 値を 1 度しか使わない (Rust の所有権モデル)。
- 効果システム (effect system): IO, 状態などを型に。
- GADT (Generalised Algebraic Data Types)。
- 型レベルプログラミング。

### 型検査の効果

未然に防げる: 型誤り、null 参照、所有権違反、並行アクセスのレース（Rust）、SQL インジェクション（型付き SQL ライブラリ）。

## 13.7 中間表現 (IR)

ターゲット非依存の表現。複数の種類:

- **AST**: 元の構造に近い。
- **3 番地コード**: `t1 = a + b; t2 = t1 * c`。
- **SSA (Static Single Assignment)**: 各変数は 1 度だけ代入。φ 関数で合流。LLVM, GCC が採用。
- **CPS (Continuation-Passing Style)**: 制御フローを継続として明示。
- **バイトコード**: JVM, .NET CIL, WebAssembly。

SSA は最適化解析（到達定義、生存解析、定数伝播）が劇的に簡単になる。

## 13.8 最適化

### 古典最適化

- 定数畳み込み: `2 + 3` → `5`。
- 共通部分式除去 (CSE)。
- デッドコード除去。
- ループ不変式の外出し。
- 強度低減 (`x * 2` → `x << 1`)。
- インライン展開。
- ピープホール最適化（窓を覗いて命令列を改良）。

### ループ最適化

- ループアンローリング。
- ループフュージョン / 分割。
- ループタイル化 (キャッシュ局所性)。
- 自動ベクトル化 (SIMD)。

### 関数間最適化

- インライン化、テールコール最適化、エスケープ解析、デバーチャライゼーション。

### プロファイル誘導最適化 (PGO) / LTO

実行統計を使って分岐予測ヒント、ホットコード優先配置。

## 13.9 コード生成

### 命令選択

IR を機械命令にマップ。木被覆、動的計画法。

### レジスタ割付

「どの変数をどのレジスタに置くか」を決める。
- グラフ彩色（生存範囲が干渉する変数は別レジスタ）。
- 線形スキャン（JIT 向き、高速）。
- スピル: レジスタが足りない場合はメモリへ。

### 命令スケジューリング

パイプラインのストールを避けるよう命令を並べ替える。

### 呼び出し規約 (Calling Convention)

引数・戻り値・スタック・レジスタの保存責任。System V ABI, Microsoft x64, AAPCS など。

## 13.10 ランタイム

### スタックフレーム

各関数呼び出しごとに作られる。フレームポインタ + スタックポインタ、ローカル変数、戻りアドレス。

### ガベージコレクション (GC)

- 参照カウント: シンプルだが循環を扱えない。Python が補助 GC で対処。
- マーク & スイープ: 到達可能性を辿る。
- コピー GC (Cheney): 半空間にコピー、断片化解消。
- 世代別 GC: 若い世代を頻繁に、年老いた世代をたまに。Java HotSpot, V8 が採用。
- 並行 GC・インクリメンタル GC: 一時停止を最小化。
- リージョン推論 / 所有権 (Rust): GC を持たずに安全。

### 例外処理

- スタック巻き戻し。
- ゼロコスト例外（DWARF テーブル）。

## 13.11 インタプリタと JIT

### インタプリタ

AST を直接評価、またはバイトコードを VM で実行。

- ツリーウォーク: 教育用、遅い。
- バイトコード VM: スタックマシン (JVM) / レジスタマシン (V8 Ignition, Lua)。

### JIT (Just-In-Time)

実行時にネイティブコードへコンパイル。
- HotSpot (Java)、V8 (JavaScript)、PyPy (Python)、TurboFan、SpiderMonkey。
- 多段最適化: ベースライン JIT → 最適化 JIT。
- インラインキャッシュ、型フィードバック、脱最適化（仮定が崩れたらインタプリタに戻る）。

### AOT (Ahead-of-Time)

ビルド時にコンパイル。GraalVM, .NET Native, Dart AOT。

## 13.12 ドメイン特化コンパイラ

- データベースのクエリオプティマイザ。
- 機械学習の XLA, TVM, MLIR（モデル → ハードウェア向け最適コード）。
- 正規表現エンジン。
- Verilog/VHDL の合成。
- シェーダコンパイラ (HLSL, GLSL, MSL → SPIR-V → ドライバ最適化)。

## 13.13 実装プロジェクトのおすすめ

1. 電卓（字句 → 構文 → 評価）。
2. Tiny BASIC または Scheme 風言語。
3. Lox（Crafting Interpreters の言語）。
4. 静的型付き ML 風言語に Hindley-Milner。
5. LLVM IR を出力するフロントエンド。
6. WebAssembly コンパイラ。

## 13.14 演習

1. 算術式 `1 + 2 * 3 - 4` を再帰下降で解析し、AST を書け。
2. `(a + b) * (a - b)` の SSA 形式を書け。
3. 共通部分式除去で消える計算を、簡単な C 風コードで例示せよ。
4. グラフ彩色法でレジスタ割付を行う際の干渉グラフを 4 変数で描け。
5. 末尾呼び出し最適化が再帰のスタック使用を $O(1)$ にする理由を説明せよ。
6. JIT が「型フィードバック」で性能を上げる仕組みを述べよ。
7. ガベージコレクションのマーク&スイープと世代別 GC の長所短所を比較せよ。

## 13.15 まとめ

コンパイラは「形を変えても意味を保つ」変換器の集合。字句 → 構文 → 意味 → IR → 最適化 → 機械語というパイプラインは、SQL オプティマイザや ML フレームワークの計算グラフ最適化と同形。型システムは「コードに数学を埋め込む」装置で、安全性と表現力を両立させる現代の主戦場。一度作って動かすと、抽象が一気に身体化する。
