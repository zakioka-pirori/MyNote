# 第 9 章 コンピュータアーキテクチャ

## 学習目標

- 数の表現（2 進・2 の補数・浮動小数点）と論理回路の組み立てを理解する。
- CPU の構成（ALU・レジスタ・データパス・制御）を辿り、命令がどう実行されるか説明できる。
- パイプライン・キャッシュ・仮想記憶など、現代 CPU の高速化機構を体系的に把握する。
- 並列性（命令レベル・スレッドレベル・データレベル）の階層を理解する。

## 9.1 なぜアーキテクチャか

ハードウェアを知らないプログラマは、性能チューニングや低レイヤのバグで詰まる。一方で、「メモリは平面ではない」「キャッシュミスが命取り」「分岐予測ミスは数十サイクル」という事実を体感していると、コードの書き方が変わる。OS・ネットワーク・コンパイラの議論はすべてアーキテクチャの上に乗っている。

## 9.2 数の表現

### 整数

- 符号なし $n$ ビット: $0$ から $2^n - 1$。
- 2 の補数: 符号あり $n$ ビット、$-2^{n-1}$ から $2^{n-1} - 1$。負数は「ビット反転 + 1」。
  - 利点: 加算回路が同じで済む。0 が一意。
- オーバーフロー: 加算結果が表現範囲を超えると桁が落ちる。CF/OF フラグで検出。

### 浮動小数点 (IEEE 754)

$\pm m \times 2^e$ の形。32 ビット (`float`) と 64 ビット (`double`) がある。

- 符号 1 ビット
- 指数（バイアス付き）8 / 11 ビット
- 仮数 23 / 52 ビット

特殊値: $\pm 0$, $\pm\infty$, NaN、非正規化数。

注意点:
- `0.1 + 0.2 != 0.3`（10 進が 2 進で循環）。
- 桁落ち: 近い値の引き算で精度が落ちる。
- 等値比較を `==` でなく $|a-b| < \varepsilon$ で。

## 9.3 ブール代数と論理回路

### ゲート

- AND, OR, NOT, XOR, NAND, NOR。
- NAND だけで全結合子を表現できる（完全集合）。

### 組合せ回路

入力だけで出力が決まる。

代表回路:
- 半加算器（HA）: 1 ビット加算 + 桁上がり。
- 全加算器（FA）: 桁上がり入力も考慮。
- 多重化器 (MUX): セレクタで入力を選ぶ。
- デコーダ・エンコーダ。
- 算術論理ユニット (ALU): 加算・減算・論理演算をまとめたもの。

### 順序回路

状態を持つ回路。フリップフロップ（D, JK, T）が基本素子。

- レジスタ: 複数 D-FF の集合。
- カウンタ。
- 有限状態機械 (FSM): 入出力 + 状態遷移。

## 9.4 命令セットアーキテクチャ (ISA)

ハードウェアとソフトウェアの境界。代表的な ISA:
- x86 / x86-64: CISC、複雑命令、可変長。歴史的経緯で巨大。
- ARM: RISC、固定長 (32/64 bit)、モバイルとサーバ。
- RISC-V: オープンな RISC、教育・研究で人気。

### 命令の種類

- データ転送: load, store, mov。
- 算術論理: add, sub, and, or, shift。
- 制御: branch, jump, call, return。
- システム: syscall, interrupt。

### アドレッシングモード

直接、レジスタ間接、ベース + オフセット、インデックス、相対。

### アセンブリ例（RISC-V）

```asm
addi x10, x0, 5    # x10 = 5
addi x11, x0, 3    # x11 = 3
add  x12, x10, x11 # x12 = 8
```

## 9.5 CPU データパス

単一サイクル CPU の 5 ステージ:

1. **IF (Instruction Fetch)**: PC から命令を取得。
2. **ID (Instruction Decode)**: 命令を解読、レジスタ読み出し。
3. **EX (Execute)**: ALU で計算。
4. **MEM (Memory Access)**: load/store ならメモリアクセス。
5. **WB (Write Back)**: 結果をレジスタに書き戻す。

制御ユニットが各ステージの信号を統括する。

## 9.6 パイプライン

5 ステージを重ねて実行することで、理想的にはサイクルあたり 1 命令 (CPI = 1) を達成。

### ハザード

- **構造ハザード**: 同じハードウェアの取り合い（命令メモリとデータメモリを分ければ解消）。
- **データハザード**: 後続命令が前命令の結果に依存。
  - フォワーディング（バイパス）。
  - パイプラインストール。
  - load-use の 1 サイクル遅延。
- **制御ハザード**: 分岐命令でフェッチが無駄に。
  - 分岐予測（静的・動的・グローバル履歴・パーセプトロン分岐予測）。
  - 投機実行。

### スーパースカラ・OoO 実行

複数の実行ユニットを並列に動かす。命令を順序通り発行し（in-order issue）、依存のない命令を **アウトオブオーダ** で実行し、結果は順序通りに退役（in-order retire）。命令ウィンドウ、Tomasulo アルゴリズム、レジスタリネーミング。

### Spectre / Meltdown

投機実行を悪用したサイドチャネル攻撃。アーキテクチャの最適化が **セキュリティ脆弱性** を生むことを示した。詳細は第 15 章。

## 9.7 メモリ階層

CPU 速度はメモリ速度より遥かに速い。階層化で平均アクセス時間を縮める。

| 層 | 容量 | 遅延 |
|---|---|---|
| レジスタ | 数 KB | 1 サイクル |
| L1 キャッシュ | 数十 KB | 4 サイクル |
| L2 キャッシュ | 数百 KB | 12 サイクル |
| L3 キャッシュ | 数十 MB | 40 サイクル |
| 主記憶 (DRAM) | GB | 100+ サイクル |
| SSD | TB | 数万サイクル |
| HDD | TB | 数百万サイクル |

### 局所性

- **時間局所性**: 一度使ったデータは再利用される。
- **空間局所性**: 近くのデータも使われる。

ループのアクセスパターン、構造体の配置、配列の走査順がキャッシュ効率を決める。

### キャッシュ構造

- ライン (64 B が一般)。
- 直接マップ・セットアソシアティブ・フルアソシアティブ。
- 置換ポリシ: LRU, FIFO, ランダム。
- 書き込み: ライトスルー / ライトバック、ライトアロケート / ノーアロケート。
- ヒット率・ミス率・MPKI。

### キャッシュコヒーレンス

マルチコアで各コアの L1 キャッシュが矛盾しないようにする MESI プロトコル等。

## 9.8 仮想記憶

各プロセスに独立したアドレス空間を提供する。

- ページング: 4 KB ページ単位、ページテーブルで仮想 → 物理を翻訳。
- 多段ページテーブル: x86-64 は 4 段（PML4 → PDPT → PD → PT）。
- TLB (Translation Lookaside Buffer): ページテーブルのキャッシュ。
- ページフォルト: ページが物理にないとき OS が処理。
- スワップ: 物理メモリ不足時にディスクに退避。

仮想記憶は OS とハードウェアの協調で動く。詳細は第 10 章。

## 9.9 I/O とバス

- メモリマップド I/O: デバイスをメモリアドレスとして扱う。
- ポート I/O: 専用命令。
- 割り込み: デバイスから CPU への通知。割り込みベクタ、割り込みハンドラ。
- DMA: CPU を介さずデバイスとメモリで転送。

バス: PCIe、USB、SATA。コヒーレントな共有メモリ vs 非コヒーレント。

## 9.10 並列性の階層

- **命令レベル並列 (ILP)**: パイプライン、スーパースカラ、VLIW。
- **データ並列**: SIMD（SSE, AVX, NEON）、GPU。
- **スレッドレベル並列**: マルチコア、SMT (Hyper-Threading)。
- **要求レベル並列**: クラスタ、データセンタ。

Amdahl の法則: 直列部分 $s$ がボトルネック。$N$ プロセッサでの最大スピードアップは $1 / (s + (1-s)/N)$。

Gustafson の法則: 問題サイズも増やせばスピードアップは頭打ちにならない。

## 9.11 GPU と特殊アクセラレータ

- GPU: 大量の単純コアでデータ並列。SIMT モデル（CUDA, OpenCL）。
- TPU: 行列積に特化（Google）。
- FPGA: 再構成可能ロジック。
- ASIC: 特定用途専用 (例: Bitcoin マイニング)。

機械学習の発展とアクセラレータの進化は表裏一体。

## 9.12 性能の測り方

- スループット (IPC, FLOPS) と遅延 (latency)。
- ベンチマーク: SPEC, MLPerf。
- プロファイリング: `perf`, VTune, gprof。
- ルーフラインモデル: 計算強度 vs メモリ帯域。

## 9.13 演習

1. $-37$ を 8 ビット 2 の補数で表せ。
2. `0.1 + 0.2 != 0.3` を IEEE 754 表現で説明せよ。
3. NAND ゲート 4 個で XOR を構成せよ。
4. RISC-V で `for (i = 0; i < 10; i++) sum += a[i];` をアセンブリで書け。
5. データハザードのフォワーディングが効かないケース（load-use）を例示せよ。
6. キャッシュサイズ 32 KB、ライン 64 B、4-way の場合のセット数を計算せよ。
7. `for (i; i<N; i++) for (j; j<N; j++) A[j][i]` と `A[i][j]` のキャッシュ効率の違いを説明せよ。
8. Amdahl の法則で、直列 10% の作業を 100 コアで実行したときのスピードアップを求めよ。

## 9.14 まとめ

コンピュータアーキテクチャは「命令と数とビットがどう物理を動かすか」の物語。トランジスタ → 論理回路 → ALU → CPU → メモリ階層 → 並列という階層を辿ると、ソフトウェアの全レイヤが地に足の着いたものになる。プログラムが速いか遅いかの 9 割はキャッシュとブランチに左右される。アーキテクチャの感覚を持ったエンジニアは、机上の計算量解析に加えて「実機の挙動」を見抜ける。
