# 第 19 章 コンピュータグラフィックス

## 学習目標

- 2D / 3D グラフィックスの数学的基盤（座標変換・ベクトル・行列）と幾何の処理を理解する。
- ラスタライズとレイトレーシングという 2 大レンダリングパラダイムを把握する。
- シェーダ・ライティング・テクスチャ・アニメーションといった現代 GPU パイプラインを学ぶ。

## 19.1 なぜグラフィックスを学ぶか

ゲーム, VR/AR, 動画編集, シミュレーション, データ可視化, 機械学習の入力として現代のあらゆる計算は最終的に画像を生成する。GPU プログラミングは AI でも必須となった。グラフィックスは線形代数と微積分の生きた応用例で、数学が「絵」になる体験は理解を加速する。

## 19.2 画像の基礎

- ピクセル, 解像度。
- 色空間: RGB, HSV, CMYK, sRGB, Linear RGB, P3, Rec.2020。
- ガンマ補正（人間の知覚は線形でない）。
- アルファチャンネル, 合成 (Porter-Duff)。
- ファイル形式: PNG (可逆), JPEG (非可逆), WebP, AVIF, SVG (ベクター)。

## 19.3 2D グラフィックス

### ラスター描画

- ライン: Bresenham のアルゴリズム。
- 円・楕円。
- 多角形フィル, スキャンライン。
- アンチエイリアス（サブピクセル, MSAA）。

### ベクター

- ベジエ曲線（2 次・3 次）。
- B スプライン, NURBS。
- パスとストローク。
- フォントレンダリング（ヒンティング, サブピクセル）。

### 2D 変換

平行移動 + 回転 + スケール + 剪断。同次座標で 3×3 行列にまとめると合成・逆が容易。

## 19.4 3D の基礎

### 座標系

- ワールド座標, モデル座標, ビュー（カメラ）座標, クリップ座標, スクリーン座標。
- 右手系 / 左手系。

### 同次座標と 4×4 行列

3D 変換を 4×4 行列で統一表現。`gl_Position = Projection * View * Model * vec4(pos, 1.0)` の形。

### 投影

- 平行投影 (Orthographic): 工学図面。
- 透視投影 (Perspective): 視点から遠いほど小さく見える。

## 19.5 ジオメトリ表現

- メッシュ: 頂点 + インデックス + 法線 + UV。
- 暗黙曲面 (SDF): 関数の等値面。レイマーチング。
- ボリューム: ボクセル, 医療画像。
- ポイントクラウド: LiDAR, 3D スキャン。
- 細分曲面 (Catmull-Clark)。
- 半辺データ構造 (half-edge)。

## 19.6 レンダリングパイプライン (ラスタライズ)

GPU の固定 + プログラマブル段階:
1. Vertex Shader: 頂点ごとに座標変換。
2. (オプション) Tessellation, Geometry Shader。
3. ラスタライズ: 三角形をピクセルに変換。
4. Fragment Shader: ピクセルごとに色を決定。
5. 出力結合 (Blend, Z-test)。

### 隠面消去

- Z バッファ: 深度値で上書き。
- ペインター法: 後ろから描く。
- カリング (背面・視錐台・遮蔽)。

### シェーダ言語

GLSL, HLSL, MSL, WGSL。GPU 並列で SIMT 実行。

## 19.7 ライティング

### Phong モデル

光 = 環境光 + 拡散反射 + 鏡面反射:
$$I = k_a I_a + k_d (\mathbf{l} \cdot \mathbf{n}) I_d + k_s (\mathbf{r} \cdot \mathbf{v})^\alpha I_s$$

### Blinn-Phong

ハーフベクトルで効率化。

### 物理ベースレンダリング (PBR)

物理法則に近づけた材質モデル。BRDF (Bidirectional Reflectance Distribution Function), GGX, Cook-Torrance。
- アルベド, ラフネス, メタリック, 法線マップ, AO。
- HDR, トーンマッピング。

### グローバルイルミネーション (GI)

- レイトレーシング, パストレーシング。
- 放射輝度伝達 (Radiosity)。
- スクリーンスペース近似 (SSAO, SSR)。
- リアルタイム GI: ボクセルコーン, RTX, Lumen。

## 19.8 テクスチャ

- テクスチャ座標 (UV)。
- フィルタリング: 最近傍, 双線形, 三線形, 異方性。
- ミップマップ。
- バンプマップ, 法線マップ, ディスプレースメント。
- テクスチャアトラス。

## 19.9 レイトレーシング

各ピクセルから光線を投げ, 物体との交差を計算し, 反射・屈折・散乱を再帰的に追う。

- 加速構造: BVH (Bounding Volume Hierarchy), kd 木。
- 最近傍交差 + シャドーレイ。
- パストレーシング: モンテカルロで積分し, 物理的に正確に。
- 双方向パストレーシング, MLT。
- ノイズ除去 (デノイズ): AI ベースが主流。

## 19.10 アニメーション

### キーフレーム + 補間

線形, 球面線形 (slerp), Bezier。

### スケルトンアニメーション

ボーン階層 + スキニング。線形ブレンドスキニング (LBS), デュアルクォータニオン。

### 物理ベース

- リジッドボディ: 衝突判定 (GJK, EPA), 拘束ソルバ (Position-Based Dynamics)。
- 布, 流体, パーティクル。
- 時間積分: Verlet, RK4。

### 逆運動学 (IK)

末端の目標から関節角を逆算。CCD, FABRIK, ヤコビアン法。

## 19.11 GPU プログラミング

### CUDA / Compute Shader

- スレッドブロック, グリッド, ワープ。
- 共有メモリ, アトミック。
- メモリ階層を意識した最適化。
- 機械学習との接続: テンソルコア, FlashAttention。

### Vulkan / Metal / DirectX 12 / WebGPU

低レベル API。明示的な同期, レンダーパス, パイプラインステート。

## 19.12 計算機幾何

- 凸包: Graham scan, QuickHull。
- ボロノイ図 / ドロネー三角形分割。
- 衝突判定。
- 多角形クリッピング (Sutherland-Hodgman)。
- 直線・線分交差。

## 19.13 リアルタイム vs オフライン

- リアルタイム (>= 30/60 FPS): ゲーム, AR/VR。最適化と近似が要。
- オフライン (映画 CG): 物理的正確さ, 1 フレーム数時間も許容。Pixar の RenderMan, Arnold。

## 19.14 GPGPU と機械学習との関係

GPU の SIMT は密行列演算に最適。CUDA → cuDNN → PyTorch のスタックが現在の主流。グラフィックスと AI の境界はもはや曖昧 (NeRF, 拡散モデル, Differentiable Rendering)。

## 19.15 演習

1. 2D で点を原点周りに 45° 回転 + (3, 0) 移動する 3×3 同次行列を書け。
2. 透視投影行列の各成分が表す意味を述べよ。
3. Phong モデルで拡散反射が「光と法線の内積」に比例する理由を説明せよ。
4. BVH を用いるとレイトレーシングの計算量が $O(\log n)$ になる理由を述べよ。
5. ガンマ補正なしで合成すると暗い色が濃く見える理由を、線形と非線形空間の混同で説明せよ。
6. シャドーマップ法でジャギーが出る原因と緩和策（PCF, Cascaded SM）を挙げよ。
7. リジッドボディ衝突解決で「貫通」を防ぐ Position-Based Dynamics の発想を説明せよ。

## 19.16 まとめ

コンピュータグラフィックスは数学の応用, 並列計算の最前線, AI との融合領域。座標変換と行列, シェーダパイプライン, ライティング, レイトレーシング――これらを押さえれば, ゲームエンジンから機械学習の GPU プログラミングまで地続きで理解できる。「絵が出る」という即時フィードバックは学習を加速する最大の友。
